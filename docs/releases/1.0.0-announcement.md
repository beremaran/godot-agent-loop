# Announcement draft — Godot Agent Loop 1.0

Other integrations give agents tools. **Godot Agent Loop gives them a tested
feedback loop to author, run, observe, playtest, and independently verify Godot
games.**

Watch the 65-second proof first: agents can start from an empty directory while
the Godot editor follows live activity, a human retains Pause/Resume control,
and the running game is checked through UI, logs, input, rendered screenshots,
and bounded compound assertions.

- Demo: `https://github.com/beremaran/godot-agent-loop/releases/tag/v1.0.0`
- Exact prompt, project, model, elapsed time, and replay:
  `https://github.com/beremaran/godot-agent-loop/blob/v1.0.0/docs/launch/launch-evidence.md`
- Install: `npx -y @beremaran/godot-agent-loop@1.0.0`

The release ships 167/167 E2E-tested tools and 358 traced actions behind a
39-tool default surface that is 81.56% smaller by schema bytes. The full Godot
4.7 candidate passed 201 MCP-to-engine tests; Godot 4.4 is the compatibility
floor. Reflection, code execution, and networking privileges remain denied by
default.

Claude Code, Codex, OpenCode, Pi, and the optional Godot Asset Library bridge all
consume the same 1.0.0 server and canonical workflow bundle. The AssetLib addon
is optional: npm/MCP-only use still installs and cleans up its transient editor
bridge automatically.

Godot Agent Loop preserves the full history and MIT attribution of
Coding-Solo/godot-mcp and tugcantopaloglu/godot-mcp. This draft is published
only after every public artifact resolves and passes its clean-install smoke.
