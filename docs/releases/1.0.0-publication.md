# Godot Agent Loop 1.0 publication playbook

Status: **prepared, not authorized, not published**. Every command in this file
changes public state unless it is explicitly labeled read-only. Execute each
wave only after the user approves that wave and the preceding verification is
green.

The formats and commands were rechecked on 2026-07-14 against the official
[GitHub release documentation](https://docs.github.com/en/repositories/releasing-projects-on-github/about-releases),

## Preflight — local only

1. Require a clean candidate commit and run `npm run check`, `npm run
   test:godot`, and `npm run test:e2e` on Godot 4.7.
2. Run the applicable Godot 4.4 floor suite and verify the 4.4/4.7 exact-tag CI
   matrix is green.
3. Build `npm run release:candidate -- <candidate-commit>` twice and compare
   `SHA256SUMS`.
4. Inspect the tarball, clean-install it, and rerun the Claude/Codex/OpenCode/Pi
   native acceptance against the exact candidate.
5. Confirm the name is still absent from GitHub, npm, MCP Registry, and AssetLib.

The local Ed25519 key can sign the tag, but GitHub currently exposes no SSH
signing key for `beremaran`. Adding that public key as a GitHub **signing key**
is part of Wave 1 approval; no global Git configuration is required.

## Wave 1 — repository, signing key, and candidate tag

Approval creates a new public repository, uploads the full history, registers a
public signing key, and publishes a signed tag. It does not modify or detach the
existing `beremaran/godot-mcp` repository.

```bash
gh repo create beremaran/godot-agent-loop \
  --public \
  --description "Evidence-first MCP automation loop for Godot 4" \
  --source=. \
  --remote=release

# Add ~/.ssh/id_ed25519.pub as an SSH signing key in GitHub account settings.

git push release main
git -c gpg.format=ssh \
  -c user.signingkey="$HOME/.ssh/id_ed25519" \
  tag -s v1.0.0 <candidate-commit> -m "Godot Agent Loop 1.0.0"
git push release v1.0.0
```

Wait for every exact-tag job in `godot-integration.yml`. Then build and inspect
the candidate from a clean `v1.0.0` checkout.

## Wave 2 — GitHub release and npm

Approval publishes immutable source/release metadata and a public npm package.
For the first npm publication, configure the protected GitHub `npm` environment
and `NPM_TOKEN`; the prepared manual workflow publishes the exact tarball with
provenance. After the package exists, replace the token with npm trusted
publishing for `publish-npm.yml` and disallow long-lived publish tokens.

```bash
gh release create v1.0.0 dist/release/* \
  --repo beremaran/godot-agent-loop \
  --verify-tag \
  --title "Godot Agent Loop 1.0.0" \
  --notes-file docs/releases/1.0.0.md

# In GitHub Actions, manually run “Publish npm release” with:
# tag=v1.0.0
# confirmation=publish-@beremaran/godot-agent-loop@1.0.0
```

Verify from a clean directory:

```bash
npm view @beremaran/godot-agent-loop@1.0.0 --json
npx -y @beremaran/godot-agent-loop@1.0.0 setup opencode
```

## Wave 3 — MCP and client distributions

Approval writes MCP Registry metadata and makes the public repository
marketplaces installable. npm must resolve first because the MCP Registry
validates the installation package.

```bash
mcp-publisher login github
mcp-publisher publish
curl "https://registry.modelcontextprotocol.io/v0.1/servers?search=io.github.beremaran/godot-agent-loop"

/plugin marketplace add beremaran/godot-agent-loop
/plugin install godot-agent-loop@godot-agent-loop

codex plugin marketplace add https://github.com/beremaran/godot-agent-loop
codex plugin add godot-agent-loop@godot-agent-loop

pi install git:github.com/beremaran/godot-agent-loop
pi install npm:@beremaran/godot-agent-loop
```

Repeat the 39-tool handshake, one real Godot call, hidden-tool discovery, skill
inventory, update/disable/reload/uninstall, and teardown checks in isolated
client homes. Restart ChatGPT desktop and verify the public Codex marketplace
copy rather than the prior local cache.

## Wave 4 — AssetLib

Approval creates and pushes a dedicated child commit of `v1.0.0`. Its only
change is the root export attributes, so GitHub's commit archive contains only
the addon while raw icon/preview URLs remain commit-pinned.

```bash
git worktree add /tmp/godot-agent-loop-assetlib v1.0.0
cp distribution/assetlib/.gitattributes /tmp/godot-agent-loop-assetlib/.gitattributes
git -C /tmp/godot-agent-loop-assetlib add .gitattributes
git -C /tmp/godot-agent-loop-assetlib commit -m "chore: prepare AssetLib 1.0.0 download"
ASSETLIB_COMMIT=$(git -C /tmp/godot-agent-loop-assetlib rev-parse HEAD)
git -C /tmp/godot-agent-loop-assetlib archive "$ASSETLIB_COMMIT" -o /tmp/assetlib.zip
node scripts/build-assetlib-submission.js "$ASSETLIB_COMMIT"
git -C /tmp/godot-agent-loop-assetlib push release HEAD:refs/heads/assetlib-v1.0.0
```

Compare `/tmp/assetlib.zip` byte-for-byte by entry against the tested
`dist/release/godot-agent-loop-1.0.0.zip`, then submit the generated payload in
the official Asset Library account. Record every reviewer response in
`TODO.md`; do not mix sponsorship or paid support into the free/open listing.
